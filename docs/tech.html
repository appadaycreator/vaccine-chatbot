利用技術・システム構成

vaccine-chatbot の技術スタックと役割（運用前提も含む）

チャットUI 利用技術

全体像

ローカルLLM（Ollama）と RAG（PDF→ベクトル検索）を組み合わせ、厚労省資料に基づく回答を返すプロトタイプです。 フロント（GitHub Pages）からは、HTTPS で公開した API（FastAPI）に対して /chat を呼び出します。

バックエンド（API）

  • Python: 3.10+（推奨: 3.13）
  • FastAPI: HTTP API（GET /status, POST /chat など）
  • uvicorn: ASGI サーバー（開発は --reload）
  • CORS: GitHub Pages からのアクセスを許可（現状は allow_origins=["*"]）

APIバージョン: 0.2.0
API の主なエンドポイント: /health, /status, /diagnostics, /sources, /reload, /search, /generate, /chat

LLM / Embedding（ローカル推論）

  • Ollama: ローカルLLM実行基盤（生成と embedding の両方）
  • 生成モデル（例）: gemma2:2b, gemma2, llama3.1
  • Embeddingモデル: nomic-embed-text

初回の遅さ対策として、API起動時に embedding を軽くウォームアップします（失敗しても API は落としません）。

RAG（PDF取り込み・検索）

  • LangChain: PDF読み込み/分割、ベクトル検索の統合
  • PyPDFLoader: PDF→テキスト抽出（スキャンPDFはOCRが必要な場合あり）
  • RecursiveCharacterTextSplitter: チャンク分割
  • ChromaDB: ベクトルDB（永続化ディレクトリ: 既定 ./chroma_db）

PDFは ./pdfs（環境変数 PDF_DIR）に配置します。追加/更新後は POST /reload で再インデックスできます。 スキャンPDFでテキスト抽出できない場合は「要OCR」として案内されます。

フロントエンド（GitHub Pages）

  • docs/index.html + docs/app.js + docs/style.css: 静的チャットUI
  • ブラウザから POST /chat を呼び出し、根拠（参照ページ）や timings を表示
  • API Base URL は localStorage または ?api=... で指定
  • 送信中の多重実行防止、再送、入力履歴、Markdown表示、エラー要点＋詳細ログ（折りたたみ）を実装

公開/運用（例）

  • cloudflared: ローカルAPIを HTTPS で公開（GitHub Pages から Mixed Content にならないようにする）
  • launchd（macOS）: uvicorn をログイン/再起動後に自動起動（テンプレ: launchd/com.vaccine.api.plist）

Quick Tunnel はURLが変わるため、常時稼働は named tunnel が推奨です（README参照）。
